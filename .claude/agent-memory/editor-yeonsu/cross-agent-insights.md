## From Novel Writer (박서준) - Editor Impact

### Human Author Technique Extraction (5 techniques)

1. **Camera Pan Sequence**: 캐릭터 소개 = 공간 -> 외양 -> 행동 -> 감각 순서
   - AI는 행동/대사로 시작하는 경향
   - 편집 시 캐릭터 첫 등장 순서 점검

2. **Small Movement Before Big Emotion**: 큰 감정 직전에 "사물의 작은 움직임" 배치
   - 바람, 먼지, 풀잎 = 정적 속 에너지 축적
   - 01-02-02 "불꽃도 꽃이잖아!" 직전에 정적 1-2문장 추가 필요

3. **Narrator Yields to Character**: "~이 분명했다" 서술자 판단 -> 캐릭터 행동으로
   - 서술자가 캐릭터보다 먼저 판단하면 안 됨
   - 기존 "tell 삭제" 규칙의 세분화: "서술자 선제 판단형 tell" = 최우선 삭제

4. **Dialogue Length as Character Fingerprint**: 대사 길이/유형의 일관성
   - 수진=중간길이 완전문장, 민준=짧은 감탄/질문, 지훈=사실적 관찰
   - 편집 시 대사 길이 패턴 점검 추가

5. **One-Sentence Showing**: "차체가 미세하게 기울었다" = 설정을 1문장으로 show
   - AI는 설정 showing에 3-5문장 사용 경향
   - 편집 시 "이 설정을 1문장 show로 압축할 수 있는가?" 테스트

### "그때"/"바로 그때" Usage Rule
- 인간 확정본: 4회/3편 (물리적 사건 돌발에만)
- AI 작성본: 11회/4편 (분위기 전환, 아이디어 발생에도)
- **Rule**: "그때"/"바로 그때" = 물리적 사건 돌발에만 사용. 분위기 전환은 감각 변화로 처리

### Dialogue Density Rule
- 연속 대사 4회 이상 -> 중간에 행동/감각/정적 삽입
- 01-02-04 M3-B3 결심 장면: 대사 6개 연속 = "대본" 느낌
- **Application**: 대사 3회 초과 시 행동 묘사 1회 이상 삽입

### Comic Beat Compression
- 인간 확정본: 코믹 비트 = 3-4문단
- AI 01-02-03 감옥 코믹: 7문단 = 과잉
- **Rule**: 코믹 비트 상한 = 5문단. 서사적 기능 있는 코미디만 유지

### M3-B1 경이의 순간 확장 필요
- 플래시 점등 = 씬의 시각적 절정인데 2문장으로 처리됨
- "플래시가 켜졌다"를 독립 문장으로 분리 + 빛의 경로 추적 묘사
- **Rule**: 비주얼 정점 = 슬로우 모션 기법 (최소 4-5문장)

### 삼인조 시너지 처리 수정
- 내 기존 규칙 "DELETE always" -> 수정 필요
- 박서준 제안: 수진 내면 회고 형태로 연결 ("아까 민준이... 방금 수진이... 지금 지훈이...")
- **Revised Rule**: 서술자 직접 해설 = 삭제. 캐릭터 내면 회고 = 허용 (초등 고학년~중학생 독자에게 연결 한 번 짚어주기)

## From Content Director (김태수) - Editor Impact

### Sentence Multi-functionality Metric
- Human avg: 1.8 functions/sentence, AI+feedback: 1.2, AI unreviewd: 1.1
- "단일 기능 문장" = 편집 시 삭제/통합 1순위 후보
- **Application**: 문장 기능 테스트 (정보/캐릭터/감정 중 1개 이상)를 편집 체크리스트에 추가

### Rule-based vs Style-based Correction
- Rule corrections (호칭, 소품): immediate effect, no regression
- Style corrections (대사 변주, 종료 방식): require repeated feedback, regression possible
- **Implication**: 편집 시 style 문제는 "규칙화 가능한 부분"과 "반복 훈련 필요 부분" 분리

### 수진 허당 비트 균형
- 1장: 쿵!+아야! = 허당의 물리적 사건
- 2장: 정자 넘어짐 1회만 = 허당 측면 희석
- **Rule**: 각 장(chapter) 내 수진의 "허당" 비트 최소 1회 보장. 편집 시 점검

### Emotion Margin (감정의 여백)
- 인간: "말하지 않는 것"으로 감정 전달 (수진이 이마 만지작거리며 삭제 지켜보기)
- AI: 여백을 설명으로 채움
- **Application**: "이 문장을 삭제하면 독자가 감정을 직접 느끼는가?" 테스트

### Discussion Scene Rhythm Variation
- 수수께끼 토론: 각 해석이 동일 리듬으로 등장 = 권태
- **Rule**: 논의/토론 장면에서 해석 제시 리듬을 변주 (첫째=길게, 둘째=짧게, 셋째=코믹하게)

### 겨울이 남쪽 갈래 복선 미완결
- 03에서 남쪽 갈래 집중 -> 화살표가 남쪽 갈래에 있다는 명시적 확인 부재
- **Application**: 복선 설치 후 "명시적 확인" 단계가 있는지 편집 시 체크

### 어르신 대사의 서사적 활용 가능성
- 02 느티나무 어르신 "보물이라도 찾는 거야?" = 현재 분위기 전달만
- Scene 문서에 없는 내용이므로 권장 사항으로 유지
