# 평가에서 발견한 패턴 모음

## Act->Chapter->Scene 공통 패턴
- **정보 손실 패턴**: 상위 문서가 A등급이어도 하위 문서가 자동으로 우수하지 않음. "독립적 재작성" vs "상위 가이드의 비트 수준 분해"
- **D항목 약진 패턴**: 1차에서 가장 낮은 항목(D)이 2차에서 가장 크게 상승하는 패턴 반복
- **재평가 D->A 도약**: Chapter +32~34점, Scene +30점. 필수 보완이 핵심 문제를 정확히 짚으면 극대화
- **피드백 전수 수용**: D등급 초안에서는 대부분의 피드백이 정당. 보류/거절은 B등급 이상에서 발생

## 초안 문서 공통 결함 세트
- Act: 공간 동선/시간 프레임 누락, 성장 아크 0점, 유머 0점, 연령 적합성 0점, 독자 경험 부재
- Scene: (1) 비트별 감정 태그 부재, (2) 전환 트리거 부재, (3) 페이싱 가이드 부재, (4) 참조 문서 부재, (5) 비트 기능 라벨 부재

## Chapter 고유 패턴
- 모멘트 구조화 패턴: 단일 씬에서 5개 모멘트에 비중%/긴장도/캐릭터/전환트리거/감각/유머 부여 -> 모범 사례
- 인지 부하 관리 3요소: 캐릭터 단계적 투입, 태그 행동으로 기억 보조, 핵심 정보 다관점 반복
- "분량이 큰 장 = 가이드도 풍부해야" 원칙
- 4개 씬 > 단일 씬 (D/A항목에서 유리)
- 미니 클라이맥스 씬 경계 문제: 정점을 하나의 씬에 명확히 귀속

## Scene 고유 패턴
- **"대사 나열" 함정**: 비트를 대사 교환으로만 구성 -> 비트 기능 다양화 필수 (6종: 사건/정보/대사/유머/상호작용/감정전환)
- **비트 단위 감정 추적**: B-2가 가장 결정적 감점 요인. 비트마다 감정 키워드 1개 필수
- **3중 태그 체계**: **감정**(키워드) + [감각:] + [기능 라벨] = Scene 문서의 표준
- **전환 설계의 복합 구조**: "서사적 계기 + 감각적 변화"가 기본 패턴
- **감정선 조감도**: 비트/모멘트 구조의 상위 요약. Scene 문서 작성 시 필수
- **V커브 감정 구조**: 기대->하강->바닥->급상승->정점. "시행착오" 씬에서 재사용 가능
- **전환 트리거 캐릭터 일치**: Chapter 배정과 Scene 배정 불일치 시 구조적 결함
- **시간 정합성**: Chapter 시간 배정과 Scene 시간이 45분 불일치 사례 발생
- **종료 후크 누락의 다항목 파급**: 5개 항목 동시 영향
- **"막의 마지막 씬"에서 종료 후크는 최우선 검증 대상**
- **"갑자기"는 인과 트리거 부재의 신호**
- **이전 씬 종료 훅 -> 다음 씬 시작 연결 필수**: 개정 시점 차이로 단절 발생 가능

## 평가 기준 관련 패턴
- A-3/B-2 이중 감점 위험: "라벨 존재 여부" vs "라벨 내용의 효과"로 구분 권고
- "therefore/but" 원칙: 비트 단위에서는 "정당한 and then" 존재
- D-6 배점 앵커 부재
- 평가 예시와 독립 평가의 차이: 11점 차이 사례. 평가 예시는 참고자료
- 동일 점수 유지 시 질적 변화 확인 필요

## 피드백 작성 패턴
- 2차 리뷰: 1차보다 짧고 핵심만. 수용 확인 + 반영 품질 + 잔여 이슈
- 재평가서: "이전 평가 대비 변경 사항 요약"으로 시작
- 피드백 응답서: 타 평가자 수용분도 "추가 반영 사항" 섹션으로 기록
- 피드백 응답 테이블 형식: 파일 끝부분 먼저 확인

## 에필로그 고유 패턴
- **에필로그도 필수 섹션 동일 적용**: "간결"과 "빈약"은 다름. 공간 동선/시간 프레임/복선 회수/유머/연령/독자경험 모두 필요
- **에필로그 전환점은 사건이어야**: "성장의 확인"(서술) vs "유물 뉴스가 TV에 나오는 순간"(사건)
- **04-act 미해결 과제의 에필로그 처리 매핑 필수**: 7건 중 완전 1, 부분 3, 미처리 3이면 문제
- **작품 마지막 이미지는 팀 차원 결정 필요**
- **두루/거울 상태가 에필로그의 핵심 감정축**: B-6-4 열린 결말의 여운에 직결
